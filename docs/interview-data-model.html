File Descriptions

Canonical Data Files

_data/interviews.yml

The authoritative source for interview records. Each interview contains:

items:
  - id: string              # Unique identifier (slug)
    title: string           # Display title
    interviewees: [string]  # List of interviewee names
    interviewer: string     # Usually "Mike Hall"
    topic: string           # Interview topic (optional)
    conference: string      # Conference name (references interview_conferences.yml)
    conference_year: int    # Year of conference (optional)
    community: string       # Community name (references interview_communities.yml)
    recorded_date: date     # When the interview was recorded
    tags: [string]          # Categorization tags
    video_assets:           # Inline video asset references
      - platform: string    # "vimeo" or "youtube"
        asset_id: string    # Platform-specific video ID

_data/video_assets.yml

Detailed metadata for each video asset. Links back to interviews via interview_id:

items:
  - interview_id: string      # Links to interviews.yml id
    platform: string          # "vimeo" or "youtube"
    asset_id: string          # Platform-specific video ID
    url: string               # Direct URL to video
    embed_url: string         # Embeddable player URL
    title_on_platform: string # Title as shown on platform
    published_date: date      # Publication date
    thumbnail: string         # Thumbnail URL
    thumbnail_local: string   # Local thumbnail path (optional)
    duration_seconds: int     # Video length
    duration_minutes: int     # Video length (rounded)
    description: string       # Video description
    transcript: string        # Inline transcript (usually empty)
    tags: [string]            # Tags
    source: string            # Source identifier (e.g., "ugtastic")

_data/transcripts.yml

Reserved for transcript data. Currently empty (items: []). Transcripts are typically stored inline in video_assets.yml or pechakucha.yml.

Lookup Tables

_data/interview_conferences.yml

Conference definitions referenced by interviews:

conferences:
  - slug: string        # URL-friendly identifier
    name: string        # Full conference name (matched by interviews.conference)
    start_date: date    # Conference start
    end_date: date      # Conference end
    location: string    # Venue location
    description: string # Conference description

_data/interview_communities.yml

Community definitions referenced by interviews:

communities:
  - slug: string        # URL-friendly identifier
    name: string        # Full community name (matched by interviews.community)
    description: string # Community description

Source Data Files

_data/youtube_sources.yml

List of YouTube playlist IDs to fetch:

playlist_ids:
  - string  # YouTube playlist ID

_data/youtube_playlists.yml

Metadata for YouTube playlists:

playlists:
  - id: string              # Playlist ID (matches youtube_sources.yml)
    title: string           # Playlist title
    slug: string            # URL-friendly identifier
    description: string     # Playlist description
    published: datetime     # Publication timestamp
    channel_title: string   # Channel name
    item_count: int         # Number of videos
    category: string        # Category (e.g., "conference")
    conference_name: string # Associated conference
    conference_year: int    # Conference year
    thumbnail: string       # Thumbnail URL

_data/youtube_videos.yml

Individual YouTube video records:

items:
  - id: string              # YouTube video ID
    playlist_id: string     # Parent playlist ID
    playlist_slug: string   # Parent playlist slug
    title: string           # Video title
    topic: string           # Topic (optional)
    interviewees: [string]  # Interviewee names
    interviewer: string     # Interviewer name
    description: string     # Video description
    published: datetime     # Publication timestamp
    position: int           # Position in playlist
    channel_title: string   # Channel name
    thumbnail: string       # Thumbnail URL
    link: string            # Direct YouTube URL
    embed_url: string       # Embeddable player URL

_data/vimeo_videos.yml

Non-interview Vimeo content (presentations, demos, etc.):

items:
  - id: int                 # Vimeo video ID
    title: string           # Video title
    link: string            # Vimeo URL
    created: datetime       # Creation timestamp
    duration_seconds: int   # Video length
    duration_minutes: int   # Video length (rounded)
    thumbnail: string       # Thumbnail URL
    tags: [string]          # Tags
    description: string     # Video description
    uploader: string        # Uploader name
    views: int              # View count
    license: string         # License type
    category: string        # Category (e.g., "oneoff")
    embed_url: string       # Embeddable player URL
    topic: string           # Topic
    people: [string]        # People featured

_data/pechakucha.yml

PechaKucha presentation records:

items:
  - id: string              # UUID
    slug: string            # URL-friendly identifier
    title: string           # Presentation title
    speaker: string         # Speaker name
    link: string            # PechaKucha URL
    created: date           # Creation date
    views: int              # View count
    categories: [string]    # Categories
    tags: [string]          # Tags
    image_url: string       # Slide image URL
    video_url: string       # Video URL
    description: string     # Description
    transcript: string      # Full transcript text

Relationships

From To Relationship Field
interviews video_assets 1:N (embedded) video_assets[]
video_assets interviews N:1 interview_id
interviews conferences N:1 conference (name match)
interviews communities N:1 community (name match)
youtube_videos youtube_playlists N:1 playlist_id
youtube_playlists youtube_sources N:1 id in playlist_ids[]

Sync Pipeline

External video sources (YouTube, Vimeo, PechaKucha) are synchronized into the canonical interviews.yml and video_assets.yml files. The sync process:

  1. Fetches playlist/video metadata from external platforms
  2. Creates/updates entries in interviews.yml with basic interview data
  3. Creates/updates entries in video_assets.yml with detailed video metadata
  4. Links records via interview_id and video_assets[] references
